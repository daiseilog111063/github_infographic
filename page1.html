Git/GitHub 入門: 基本概念とセットアップ

Part 1 学習時間: 約30分

Gitとは？

なぜバージョン管理が必要？ ファイルの変更履歴を記録することで、「いつ、誰が、何を、なぜ変更したか」を追跡でき、誤って変更した場合も過去の状態に簡単に戻せます。複数人での共同作業もスムーズになり、新しい機能を安全に試すことも可能です。

Gitは分散型バージョン管理システムです。ソフトウェア開発やドキュメント作成における変更履歴を記録・追跡します。

分散型
中央サーバーに依存せず、各開発者がローカルに完全な履歴を持つ
スナップショット
変更を差分ではなく、ファイル全体のスナップショットとして保存
高速処理
ローカルで操作するため、ネットワーク遅延がない

GitHubとは？

GitHubはGitリポジトリのホスティングサービスです。コードの共有、プロジェクト管理、ソーシャルコーディングなどの機能を提供します。

コード共有と協業
複数人でのコード共有や協業開発を簡単に実現
プロジェクト管理
Issues、Pull Requests、Projects機能でタスク管理
ソーシャルコーディング
世界中の開発者とつながり、オープンソースに貢献

ローカル環境: あなたのPC内での作業を示します。ファイルを編集し(Working Dir)、git addでステージングし、git commitでローカルリポジトリに変更を記録します。

リモートサーバー: GitHub上のリポジトリを示します。

連携:

  • git push: ローカルの変更をリモートにアップロードします。
  • git pull / fetch: リモートの変更をローカルにダウンロードします。(pullはマージも行います)
  • git clone: リモートリポジトリをローカルにコピーしてきます。(最初の取得)

基本用語解説

Gitを使う前に知っておきたい基本的な用語を覚えましょう。

リポジトリ (Repository)

プロジェクトのファイルとその変更履歴を保存する場所です。
ローカルリポジトリ: 自分のコンピュータ上のリポジトリ
リモートリポジトリ: GitHubなどのサーバー上のリポジトリ

コミット (Commit)

変更を記録する操作。スナップショットを作成します。各コミットには一意のID（ハッシュ値）が割り当てられます。

ステージングエリア (Staging Area / Index)

次のコミットに含める変更内容を準備する場所です。作業ディレクトリでの変更の中から、コミットしたいものだけを選んで git add コマンドで追加します。インデックスとも呼ばれます。

ブランチ (Branch)

独立した開発ラインを表します。メインの開発ラインから分岐して並行開発ができます。

マージ (Merge)

異なるブランチの変更を統合する操作です。

ブランチとは？

ブランチは、開発作業を分岐させて、元のバージョンに影響を与えずに新しい機能の追加やバグ修正を行うための仕組みです。複数の作業を並行して進めることができます。

graph TD subgraph MainBranch ["mainブランチ"] direction LR commitA --> commitB; commitB --> commitC; end subgraph FeatureBranch ["featureブランチ"] direction LR commitB --> featureCommit1; featureCommit1 --> featureCommit2; end style MainBranch fill:#e3f2fd,stroke:#90caf9 style FeatureBranch fill:#e8f5e9,stroke:#a5d6a7

図の説明:

  • mainブランチ: プロジェクトの安定した本線となる履歴です。
  • featureブランチ: mainブランチの特定のコミット (commitB) から分岐し、新しい機能の開発 (featureCommit1, featureCommit2) を行っています。
  • このように分岐させることで、mainブランチの安定性を保ちながら、安全に新しい開発を進めることができます。

マージとは？

マージは、分岐したブランチでの変更内容を別のブランチ（通常は元のブランチ）に取り込む操作です。これにより、並行して進められていた開発作業を統合します。

graph TD subgraph MainBranch ["mainブランチ"] direction LR commitA --> commitB; commitB --> commitC; commitC --> MergeCommit[mergeCommit]; end subgraph FeatureBranch ["featureブランチ"] direction LR commitB --> featureCommit1; featureCommit1 --> featureCommit2; end featureCommit2 --> MergeCommit; style MergeCommit fill:#f9f,stroke:#333,stroke-width:2px style MainBranch fill:#e3f2fd,stroke:#90caf9 style FeatureBranch fill:#e8f5e9,stroke:#a5d6a7

図の説明:

  • commitBからmainブランチ (→commitC) とfeatureブランチ (→featureCommit1→featureCommit2) が分岐しています。
  • マージ操作により、mainブランチの先端 (commitC) とfeatureブランチの先端 (featureCommit2) の両方の変更内容が統合され、新しい「マージコミット」(M) が作成されました。
  • これにより、featureブランチで行われた開発成果がmainブランチの履歴に統合されます。

Git & GitHubのセットアップ

1. Gitのインストール

Windows

Git for Windowsをダウンロード: https://git-scm.com/download/win

インストーラを実行し、基本的にはデフォルト設定でインストール

「Git Bash」と「Git GUI」がインストールされます

macOS

brew install git
Updating Homebrew...

または「Xcode Command Line Tools」をインストール:

xcode-select --install

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install git

2. インストールの確認

git --version
git version 2.33.0

初めてのGit設定では、ユーザー名とメールアドレスを設定しておくと良いでしょう。

git config --global user.name "あなたの名前"
git config --global user.email "あなたのメール"

GitHubアカウントの作成

GitHub登録ページにアクセス

ユーザー名、メールアドレス、パスワードを入力

アカウント確認のステップを完了

2要素認証の設定（推奨）

GitHubへの接続にはHTTPSまたはSSH認証を使用できます。初心者にはHTTPSがお勧めです。より安全な接続にはSSH鍵を設定しましょう。

SSH鍵の設定 (オプション)

ssh-keygen -t ed25519 -C "your_email@example.com"
Generating public/private ed25519 key pair.
cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... your_email@example.com

表示された公開鍵をコピーして、GitHubの設定画面の「SSH and GPG keys」セクションに追加します。

Git GUI・統合開発環境

コマンドラインが苦手な方は、以下のツールを使うと視覚的にGitを操作できます。

ツール名 特徴 プラットフォーム
GitHub Desktop シンプルで使いやすい、GitHubとの連携が強力 Windows, macOS
Sourcetree 高機能、Git Flow対応 Windows, macOS
VS Code エディタにGit機能統合 Windows, macOS, Linux
Git Kraken 美しいUI、強力なマージ機能 Windows, macOS, Linux

GUIツールは便利ですが、Gitの基本的なコマンドを理解しておくと、トラブル時の対応がスムーズです。
